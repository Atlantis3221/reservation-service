# Фича: Множественные контактные ссылки для бронирования

**Ветка:** `11-contact-links`
**Создана:** 2026-03-14
**Обновлена:** 2026-03-17
**Статус:** реализовано

## Контекст

**Зачем:** Сейчас кнопка «Забронировать» на фронтенде ведёт только в Telegram (по `telegram_username` владельца). Владельцы бань хотят предлагать клиентам выбор мессенджера: Telegram, ВКонтакте, MAX. У каждого бизнеса может быть своя комбинация ссылок.

**Что есть сейчас:**
- В таблице `businesses` хранится `telegram_username` — единственный контакт владельца.
- Фронтенд показывает кнопку «Забронировать в Telegram», которая открывает `https://t.me/{username}`.
- Если `telegram_username` не задан — отображается текст «Свяжитесь с владельцем бани».
- `telegram_username` автоматически синхронизируется из профиля Telegram при взаимодействии с ботом.
- Реализована **админ-панель** (спека 18) — веб-чат на `admin.slotik.tech` с теми же командами, что в боте. Команды выполняются через `services/command.ts`. Команда `/settings` (настройки) уже поддерживает изменение названия и slug через оба канала.

## User Stories

### US-1 — Владелец задаёт контактные ссылки (Приоритет: P1)

Владелец бани хочет указать ссылки для связи (Telegram, VK, MAX), чтобы клиенты видели их на странице бронирования. Управление доступно через **два канала**: Telegram-бот и админ-панель (веб-чат). Для каждого мессенджера владелец предоставляет полную ссылку.

**Почему этот приоритет:** Без возможности задать ссылки фича бесполезна.

**Как проверить независимо:** В боте — `/settings` → «Ссылки для связи». В админке — команда `настройки` → кнопка «Ссылки для связи». Добавить/удалить ссылку, убедиться что ссылки сохраняются в БД.

**Сценарии приёмки (Telegram-бот):**

1. **Дано** владелец с бизнесом, **Когда** он открывает `/settings` → «Ссылки для связи», **Тогда** бот показывает текущие ссылки и кнопки для добавления отсутствующих (Telegram / VK / MAX).
2. **Дано** владелец нажал «Добавить VK», **Когда** он отправляет ссылку (например `https://vk.me/ivan`), **Тогда** ссылка сохраняется, бот подтверждает.
3. **Дано** у бизнеса есть ссылка на VK, **Когда** владелец нажимает «Удалить VK», **Тогда** ссылка удаляется.

**Сценарии приёмки (Админ-панель):**

4. **Дано** владелец в веб-чате, **Когда** отправляет `настройки`, **Тогда** в ответе видны текущие ссылки и кнопка «Ссылки для связи».
5. **Дано** владелец нажал «Ссылки для связи» в веб-чате, **Тогда** отображаются текущие ссылки и кнопки добавления/удаления.
6. **Дано** владелец нажал «Добавить VK» в веб-чате, **Когда** он вводит ссылку, **Тогда** ссылка сохраняется, чат подтверждает.
7. **Дано** владелец нажал «Удалить VK» в веб-чате, **Тогда** ссылка удаляется.

---

### US-2 — Клиент видит кнопки только для настроенных мессенджеров (Приоритет: P1)

Клиент на странице бронирования видит кнопки только для тех мессенджеров, для которых владелец указал ссылку. Порядок кнопок фиксированный: Telegram → VK → MAX.

**Почему этот приоритет:** Это основная ценность фичи для пользователя.

**Как проверить независимо:** Открыть страницу бронирования, выбрать слот, увидеть кнопки только для настроенных мессенджеров.

**Сценарии приёмки:**

1. **Дано** у бизнеса настроены ссылки на Telegram и VK, **Когда** клиент выбирает слот, **Тогда** в панели бронирования видны две кнопки: «Забронировать в Telegram» и «Забронировать в VK».
2. **Дано** у бизнеса нет ни одной ссылки, **Когда** клиент выбирает слот, **Тогда** отображается текст «Свяжитесь с владельцем бани для бронирования».
3. **Дано** у бизнеса настроена только ссылка MAX, **Когда** клиент выбирает слот, **Тогда** отображается одна кнопка «Забронировать в MAX».

---

### US-3 — Обратная совместимость с `telegram_username` (Приоритет: P1)

Если у бизнеса есть `telegram_username`, но нет записей в таблице `contact_links` — система автоматически формирует ссылку Telegram из username и показывает кнопку как раньше.

**Почему этот приоритет:** Нельзя сломать существующее поведение для существующих бизнесов.

**Сценарии приёмки:**

1. **Дано** бизнес с `telegram_username = "ivan"` и без записей в `contact_links`, **Когда** клиент открывает страницу, **Тогда** кнопка «Забронировать в Telegram» ведёт на `https://t.me/ivan`.

## Ключевые сущности

- **ContactLink**: связь бизнеса с контактом.
  - `business_id` (FK → businesses)
  - `type`: `telegram` | `vk` | `max`
  - `url`: полная ссылка, которую предоставляет владелец (например `https://t.me/ivan`, `https://vk.me/club123`, `https://max.ru/u/...`)
  - UNIQUE constraint на `(business_id, type)` — один тип = одна ссылка

- **Порядок отображения кнопок:** Telegram → VK → MAX (фиксированный).

## Edge Cases

- Владелец пытается добавить вторую ссылку того же типа → перезаписываем существующую.
- Владелец отправляет невалидный URL (без `https://`, пустой) → бот сообщает об ошибке, просит ввести корректную ссылку.
- У бизнеса есть и `telegram_username`, и запись `telegram` в `contact_links` → приоритет у `contact_links`.
