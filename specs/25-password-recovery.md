# Фича: Восстановление пароля

**Ветка:** `25-password-recovery`
**Создана:** 2026-03-18
**Статус:** черновик

## Контекст

**Зачем:** Администратор может забыть пароль и потерять доступ к панели управления. Сейчас единственный способ сбросить пароль — написать `/reset` боту в Telegram, но на странице логина нет ссылки «Забыли пароль?», и пользователь может не знать об этой возможности. Нужен стандартный UX-паттерн: ввёл email → получил ссылку → сбросил пароль.

**Что есть сейчас:**
- Бэкенд: таблица `reset_tokens`, функция `resetPassword` в `auth.ts`, эндпоинт `POST /admin/auth/reset-password`
- Бот: команда `/reset` — генерирует токен и отправляет ссылку в Telegram
- Фронтенд (admin): страница `ResetPasswordPage.tsx` — форма ввода нового пароля по токену из URL
- На странице логина **нет** ссылки «Забыли пароль?»
- В проекте **нет** email-рассылки (SMTP не настроен)

**Решение:** Добавить отправку email через SMTP (nodemailer). Если к аккаунту привязан Telegram — отправляется и email, и Telegram-сообщение. Если не привязан — только email. Telegram-команда `/reset` продолжает работать.

## User Stories

### US-1 — Ссылка «Забыли пароль?» на странице логина (Приоритет: P1)

Администратор на странице логина видит ссылку «Забыли пароль?». По клику он попадает на страницу запроса восстановления.

**Сценарии приёмки:**

1. **Дано** страница логина, **Когда** пользователь видит форму, **Тогда** под кнопкой «Войти» есть ссылка «Забыли пароль?»
2. **Дано** страница логина, **Когда** пользователь кликает «Забыли пароль?», **Тогда** он переходит на `/forgot-password`

---

### US-2 — Запрос сброса пароля по email (Приоритет: P1)

Администратор вводит свой email на странице `/forgot-password`. Система генерирует токен сброса и отправляет ссылку на email. Если к аккаунту привязан Telegram — дублирует ссылку в Telegram.

**Сценарии приёмки:**

1. **Дано** email существует в системе, **Когда** пользователь вводит email и нажимает «Восстановить», **Тогда** на email отправляется письмо со ссылкой для сброса, на экране — сообщение «Ссылка для сброса отправлена на ваш email»
2. **Дано** email существует и привязан Telegram, **Когда** пользователь запрашивает сброс, **Тогда** ссылка отправляется и на email, и в Telegram
3. **Дано** email **не** найден в системе, **Когда** пользователь вводит email, **Тогда** показывается **то же** сообщение «Ссылка отправлена на ваш email» (не раскрываем существование аккаунтов)
4. **Дано** пользователь запрашивает сброс повторно, **Когда** прошло менее 1 минуты, **Тогда** показывается «Попробуйте через минуту» (rate limiting)

---

### US-3 — Ввод нового пароля по ссылке (Приоритет: P1)

Администратор переходит по ссылке из email/Telegram, видит форму ввода нового пароля с подтверждением.

**Сценарии приёмки:**

1. **Дано** валидный токен, **Когда** пользователь вводит новый пароль (≥8 символов) и подтверждение, **Тогда** пароль обновляется, пользователь редиректится на `/login` с сообщением «Пароль успешно изменён»
2. **Дано** истёкший/использованный токен, **Когда** пользователь открывает ссылку, **Тогда** показывается ошибка «Ссылка истекла или уже использована»
3. **Дано** пароль и подтверждение не совпадают, **Когда** пользователь нажимает «Сохранить», **Тогда** показывается ошибка «Пароли не совпадают»

---

## Ключевые сущности

- **AdminUser**: аккаунт администратора (email, password_hash, owner_chat_id)
- **ResetToken**: токен сброса пароля (token, admin_user_id, expires_at, used) — уже существует

## Edge Cases

- Множественные запросы сброса: rate limit — 1 запрос в минуту на email
- SMTP не настроен (env переменные пустые): эндпоинт возвращает 500 с понятной ошибкой в логах, пользователю — «Сервис временно недоступен»
- Бот не может отправить сообщение (Telegram заблокирован): письмо на email всё равно отправляется, ошибка Telegram логируется

## Критерии приёмки

- [ ] На странице логина есть ссылка «Забыли пароль?», ведущая на `/forgot-password`
- [ ] Страница `/forgot-password` — форма ввода email + кнопка «Восстановить»
- [ ] `POST /admin/auth/forgot-password` генерирует токен и отправляет email через SMTP
- [ ] Если привязан Telegram — дублирует ссылку в Telegram
- [ ] Письмо содержит ссылку вида `{ADMIN_URL}/reset?token={token}`
- [ ] Токен живёт 30 минут, одноразовый
- [ ] Rate limit: не чаще 1 раза в минуту на email
- [ ] Не раскрывается факт существования аккаунта (одинаковый ответ)
- [ ] Страница `/reset?token=...` — форма с полями «Новый пароль» и «Подтверждение пароля»
- [ ] Клиентская валидация: пароли совпадают, ≥8 символов
- [ ] `.env.example` обновлён SMTP-переменными
- [ ] Telegram-команда `/reset` продолжает работать
