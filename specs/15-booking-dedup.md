# Фича: Предупреждение о пересечении бронирований

**Ветка:** `15-booking-dedup`
**Создана:** 2026-03-16
**Статус:** реализовано

## Контекст

**Зачем:** При создании брони владелец не видит, что на это время уже есть другая бронь (полное или частичное пересечение). Из-за этого появляются дублирующие слоты — например, два booked-слота 14:00–18:00. Нужно перед созданием брони показывать предупреждение о пересечении, но не блокировать — владелец сам решает, создавать или нет.

**Что есть сейчас:** Функция `bookRange` в `slot.repository.ts` выполняет `INSERT` без какой-либо проверки. Обработчики бота (`handleBookingCommand`, `handleBookingRangeCommand`) вызывают `bookRange` напрямую.

## User Stories

### US-1 — Предупреждение о пересечении перед бронированием (Приоритет: P1)

Владелец отправляет команду бронирования. Если на этот день уже есть бронь, пересекающаяся по времени (полностью или частично), бот предупреждает о пересечении и спрашивает подтверждение. Если пересечений нет — бронь создаётся сразу.

**Почему этот приоритет:** Единственная user story, решает проблему.

**Как проверить независимо:** Создать бронь, затем попытаться создать пересекающуюся.

**Сценарии приёмки:**

1. **Дано** на 16.03 нет бронирований, **Когда** владелец отправляет «сегодня бронь с 14 до 18», **Тогда** бронь создаётся сразу, бот подтверждает.
2. **Дано** на 16.03 есть бронь 14:00–18:00, **Когда** владелец отправляет «сегодня бронь с 14 до 18» (точный дубль), **Тогда** бот предупреждает: «⚠️ На это время уже есть бронь: 14:00–18:00. Создать ещё одну?» с кнопками Да/Нет.
3. **Дано** на 16.03 есть бронь 14:00–18:00, **Когда** владелец бронирует 16:00–20:00 (частичное пересечение), **Тогда** бот предупреждает: «⚠️ Пересекается с существующей бронью: 14:00–18:00. Создать?» с кнопками Да/Нет.
4. **Дано** бот показал предупреждение, **Когда** владелец нажимает «Да», **Тогда** бронь создаётся.
5. **Дано** бот показал предупреждение, **Когда** владелец нажимает «Нет», **Тогда** бронь не создаётся, бот сообщает «Бронирование отменено».
6. **Дано** на 16.03 есть бронь 14:00–18:00, **Когда** владелец бронирует 18:00–22:00 (смежные, не пересекающиеся), **Тогда** бронь создаётся сразу без предупреждения.

## Ключевые сущности

- **Slot** (`slots`): строка в таблице со `start_time`, `end_time`, `status`.
- **bookRange**: функция в `slot.repository.ts`, выполняющая `INSERT` нового booked-слота.

## Edge Cases

- Несколько пересечений одновременно → перечислить все пересекающиеся брони в предупреждении.
- Бронирование через полночь → проверка пересечения учитывает `date_key` + `start_time`/`end_time`.
- Пользователь не нажимает ни Да, ни Нет → ничего не происходит, бронь не создаётся.

## Критерии приёмки

- [x] При отсутствии пересечений бронь создаётся как раньше (без лишних шагов).
- [x] При наличии пересечения бот показывает предупреждение с inline-кнопками «Да» / «Нет».
- [x] При нажатии «Да» бронь создаётся и бот подтверждает.
- [x] При нажатии «Нет» бронь не создаётся.
- [x] Предупреждение показывает время пересекающихся бронирований.
