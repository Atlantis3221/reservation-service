# Фича: Предупреждение при бронировании вне рабочего времени

**Ветка:** `17-working-hours-warning`
**Создана:** 2026-03-16
**Статус:** черновик

## Контекст

**Зачем:** Сейчас при бронировании на день без заданного расписания система молча создаёт дефолтные слоты (10:00–22:00) и бронирует. Владелец не получает обратной связи о том, что бронь создана на нерабочий день или в нерабочее время. Нужно предупреждать владельца, но разрешать бронь — и помечать этот день как рабочий.

**Что есть сейчас:**
- Рабочие часы задаются через команду «на этой/следующей неделе ПН-ПТ с 10 до 23» → создаются слоты `available`
- При бронировании (бот) если на день нет слотов — автоматически добавляются слоты 10:00–22:00 без предупреждения
- Если время бронирования выходит за рамки доступных слотов — бронь всё равно создаётся, но без предупреждения
- Фронтенд показывает только дни с `available` слотами

## User Stories

### US-1 — Предупреждение при бронировании на нерабочий день (Приоритет: P1)

Владелец бронирует через бот на день, где расписание не задано. Система предупреждает, что на этот день расписание не задано, но позволяет создать бронь. День после этого становится рабочим (видимым на фронтенде).

**Почему этот приоритет:** Основной сценарий — владелец забыл задать расписание или бронирует на внеплановый день.

**Как проверить независимо:** Не задавать расписание на среду, попытаться забронировать «в среду бронь с 14 до 18 Иванов», увидеть предупреждение, подтвердить — бронь создана.

**Сценарии приёмки:**

1. **Дано** на среду расписание не задано, **Когда** отправляю «среда бронь с 14 до 18 Иванов», **Тогда** бот показывает предупреждение «⚠️ На этот день расписание не задано. Создать бронь и добавить рабочий день?» с кнопками «Да» / «Нет»
2. **Дано** предупреждение показано, **Когда** нажимаю «Да», **Тогда** бронь создаётся, на день добавляется слот `available` (чтобы день стал видимым на фронтенде)
3. **Дано** предупреждение показано, **Когда** нажимаю «Нет», **Тогда** бронь не создаётся

---

### US-2 — Предупреждение при бронировании вне рабочего времени (Приоритет: P1)

Владелец бронирует через бот на время, которое выходит за рамки заданного рабочего расписания на этот день. Система предупреждает, но позволяет создать бронь.

**Почему этот приоритет:** Важно для адекватной обратной связи — владелец должен понимать, что он бронирует «мимо» расписания.

**Как проверить независимо:** Задать расписание 10:00–18:00, попытаться забронировать с 8:00 до 10:00 или с 19:00 до 21:00.

**Сценарии приёмки:**

1. **Дано** расписание 10:00–18:00, **Когда** бронирую с 8:00 до 10:00, **Тогда** предупреждение «⚠️ Бронь выходит за рабочее время (10:00–18:00). Создать?» + кнопки
2. **Дано** расписание 10:00–18:00, **Когда** бронирую с 12:00 до 14:00 (внутри), **Тогда** бронь создаётся без предупреждения
3. **Дано** расписание 10:00–18:00, **Когда** бронирую с 16:00 до 20:00 (частично выходит), **Тогда** предупреждение показывается

### US-3 — Тесты (Приоритет: P1)

Написать unit-тесты на всю логику определения «вне рабочего времени» и на процесс бронирования с предупреждением.

**Как проверить независимо:** Запустить `npm test`.

**Сценарии приёмки:**

1. Тест: `isOutsideWorkingHours` — нет слотов на день → true
2. Тест: `isOutsideWorkingHours` — бронь внутри рабочего времени → false
3. Тест: `isOutsideWorkingHours` — бронь частично выходит за рабочее время → true
4. Тест: `isOutsideWorkingHours` — бронь полностью вне рабочего времени → true

## Ключевые сущности

- **Slot (available)**: рабочий слот, задаёт рабочее время дня. Если `available` слотов на день нет — день нерабочий.
- **Working hours check**: новая функция `checkWorkingHours(businessId, dateKey, startTime, endTime)` — определяет, попадает ли бронь в рабочее время. Возвращает `{ isWorkDay: boolean; isWithinHours: boolean; workingRange?: string }`.

## Edge Cases

- **Несколько `available` слотов на день (с разрывом):** проверяется общий диапазон (от минимального start до максимального end).
- **Ночное заведение (рабочие часы через полночь):** если расписание 10:00–03:00, бронь 01:00–03:00 — это внутри рабочего времени. Логика учитывает переход через полночь.
- **Overlap + вне рабочего времени:** оба предупреждения объединяются в одно сообщение с одной парой кнопок «Да/Нет».
- **Фронтенд:** при подтверждении брони на нерабочий день автоматически создаётся `available` слот на весь день — день становится видимым на фронтенде. Фронтенд не меняется.

## Критерии приёмки

- [ ] Новая функция `checkWorkingHours(businessId, dateKey, startTime, endTime)` корректно определяет: рабочий ли день, попадает ли бронь в рабочее время
- [ ] Ночные смены (10:00–03:00): бронь 01:00–03:00 считается внутри рабочего времени
- [ ] При бронировании на нерабочий день — предупреждение «⚠️ На этот день расписание не задано. Создать бронь и добавить рабочий день?»
- [ ] При бронировании вне рабочего времени — предупреждение «⚠️ Бронь выходит за рабочее время (10:00–18:00). Создать?»
- [ ] Предупреждение «вне рабочего времени» и «пересечение» объединяются в одно сообщение
- [ ] При подтверждении брони на нерабочий день — автоматически создаётся `available` слот (один раз) и день становится видимым на фронтенде
- [ ] При бронировании внутри рабочего времени — бронь создаётся без предупреждения (как раньше)
- [ ] Unit-тесты на `checkWorkingHours`: нет слотов, внутри, частично снаружи, полностью снаружи, ночная смена
- [ ] Unit-тесты на логику бронирования с предупреждением
- [ ] Существующие тесты не сломаны
