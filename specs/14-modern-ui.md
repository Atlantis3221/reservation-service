# Фича: Приведение UI к современному виду

**Ветка:** `14-modern-ui`
**Создана:** 2026-03-16
**Статус:** реализовано

## Контекст

**Зачем:** Большинство пользователей заходят с телефонов. Текущий интерфейс перегружен: лишние элементы управления (выбор времени, кнопка «Сегодня»), рамка календаря со скроллом создаёт ощущение «вложенного окна», навигация между днями требует выхода в месячный вид. Нужно упростить интерфейс, сделать его чистым и удобным для мобильных устройств.

**Что есть сейчас:**
- **День (day view):** расписание отображается внутри рамки `.gcal` с border-radius, box-shadow, фоном с прозрачностью и backdrop-blur. Таймлайн `.gcal-timeline` имеет `max-height: 600px` и `overflow-y: auto` — на мобильных это создаёт вложенный скролл. Под календарём — панель бронирования `.booking-bar` с выбором времени «С» / «До» (два `<select>`) и кнопкой «Забронировать в Telegram». Нет возможности перейти к соседнему дню — только «← К календарю».
- **Месяц (month view):** в тулбаре есть кнопка «Сегодня» (`goToday`), стрелки навигации и заголовок месяца.

## User Stories

### US-1 — Чистый дневной вид без лишних элементов (Приоритет: P1)

Пользователь открывает день и видит чистое расписание на белом фоне, без рамки и вложенного скролла. Расписание помещается по высоте экрана. Внизу — только кнопка «Забронировать в Telegram» без настройки времени.

**Почему этот приоритет:** Основной экран, который видят пользователи при бронировании.

**Как проверить независимо:** Открыть день с мобильного — расписание читается без скролла внутри рамки, кнопка бронирования доступна.

**Сценарии приёмки:**

1. **Дано** пользователь на мобильном устройстве, **Когда** открывает дневной вид, **Тогда** расписание отображается на белом фоне без рамки и вложенного скролла, помещается по высоте экрана.
2. **Дано** пользователь выбрал дату, **Когда** видит панель бронирования, **Тогда** нет выбора времени «С» / «До» — только кнопка «Забронировать в Telegram».
3. **Дано** пользователь нажал «Забронировать в Telegram», **Когда** сообщение копируется, **Тогда** сообщение содержит только дату (без времени), открывается Telegram.

---

### US-2 — Быстрая навигация по дням недели (Приоритет: P1)

Пользователь находится в дневном виде и может быстро перейти к другому дню текущей недели, не возвращаясь в месячный вид.

**Почему этот приоритет:** Экономит время, уменьшает количество нажатий. Критично для мобильных.

**Как проверить независимо:** Находясь в дневном виде, переключиться на соседний день одним нажатием.

**Сценарии приёмки:**

1. **Дано** пользователь в дневном виде на понедельник, **Когда** нажимает на среду в навигации по неделе, **Тогда** открывается расписание на среду без перехода в месячный вид.
2. **Дано** пользователь в дневном виде, **Когда** смотрит навигацию по неделе, **Тогда** видит все 7 дней текущей недели с обозначением выбранного дня и дней с доступными слотами.

---

### US-3 — Убрать кнопку «Сегодня» из месячного вида (Приоритет: P2)

Кнопка «Сегодня» убирается из тулбара месячного вида — она занимает место и редко используется.

**Почему этот приоритет:** Косметическое упрощение, не блокирует остальное.

**Как проверить независимо:** Открыть месячный вид — кнопки «Сегодня» нет, навигация стрелками и заголовок месяца сохранены.

**Сценарии приёмки:**

1. **Дано** пользователь в месячном виде, **Когда** смотрит тулбар, **Тогда** кнопки «Сегодня» нет, есть только стрелки навигации и название месяца.

---

## Ключевые сущности

- **Day view (дневной вид):** экран расписания на выбранный день с таймлайном слотов
- **Week nav (навигация по неделе):** горизонтальная полоска из 7 дней текущей недели для быстрого переключения
- **Booking bar (панель бронирования):** упрощённая панель с одной кнопкой «Забронировать в Telegram»

## Edge Cases

- Что если неделя пересекает два месяца (например, Пн-Ср в марте, Чт-Вс в апреле)? Навигация по неделе должна корректно показывать дни обоих месяцев.
- Что если на некоторые дни недели нет расписания? Показывать их в навигации, но при выборе — «Расписание не задано».

## Критерии приёмки

- [ ] Дневной вид: белый фон, нет рамки `.gcal`, нет вложенного скролла, расписание помещается по высоте экрана
- [ ] Дневной вид: нет выбора времени «С» / «До», только кнопка «Забронировать в Telegram»
- [ ] Дневной вид: сообщение для Telegram содержит только дату (без времени)
- [ ] Навигация по неделе: отображается в дневном виде, позволяет переключать дни одним нажатием
- [ ] Навигация по неделе: показывает индикатор выбранного дня и дней с доступными слотами
- [ ] Месячный вид: кнопка «Сегодня» убрана
- [ ] Интерфейс удобен на мобильных устройствах (touch-friendly размеры, отсутствие вложенного скролла)
