# Фича: Бургер-меню в админке

**Ветка:** `21-admin-burger-menu`
**Создана:** 2026-03-17
**Статус:** реализовано

## Контекст

**Зачем:** Сейчас шапка админки перегружена элементами: лого «Slotik», переключатель заведений, ссылка на расписание (страница бани), кнопка привязки Telegram, email пользователя, кнопка выхода. На мобильных устройствах это выглядит тесно. Нужно разгрузить шапку — вынести второстепенные элементы в бургер-меню (боковую панель).

**Что есть сейчас:**
- Шапка (`chat-header`): слева — лого «Slotik» + `BusinessSwitcher` + ссылка на расписание бани; справа — кнопка привязки Telegram (🔗) + email + кнопка выхода.
- Привязка Telegram открывается баннером под шапкой (`link-banner`).
- Ссылка на расписание ведёт на `slotik.tech/{slug}` (страница бани для клиентов).

## User Stories

### US-1 — Бургер-меню с навигацией и настройками (Приоритет: P1)

Пользователь видит в шапке кнопку-бургер (☰). По нажатию открывается боковая панель (sidebar / drawer) с элементами, убранными из шапки:

1. **Лого «Slotik»** — отображается в верхней части бургер-меню.
2. **Привязка Telegram** — кнопка, по клику открывает форму ввода 6-значного кода (текущий компонент `LinkTelegram`).
3. **Email пользователя** — отображается информационно.
4. **Кнопка выхода** — логаут.

В шапке остаются: кнопка бургера + `BusinessSwitcher` + ссылка на расписание бани.

**Как проверить независимо:** Открыть админку, нажать кнопку-бургер, убедиться что все элементы на месте, ссылки работают.

**Сценарии приёмки:**

1. **Дано** пользователь в чате, **Когда** нажимает кнопку-бургер (☰), **Тогда** открывается боковая панель с лого, привязкой Telegram, email и кнопкой выхода.
2. **Дано** открытое бургер-меню, **Когда** нажимает «Привязать Telegram», **Тогда** появляется форма ввода кода (существующий `LinkTelegram`).
3. **Дано** открытое бургер-меню, **Когда** нажимает «Выйти», **Тогда** пользователь разлогинивается.
4. **Дано** открытое бургер-меню, **Когда** нажимает за пределами панели (или кнопку закрытия), **Тогда** панель закрывается.
5. **Дано** лого «Slotik» ранее в шапке, **Тогда** теперь лого отображается только в бургер-меню, в шапке его нет.
6. **Дано** ссылка на расписание бани и переключатель заведений, **Тогда** они остаются в шапке как и раньше.

## Ключевые сущности

- **Burger Menu (Sidebar / Drawer)**: боковая панель, выезжающая слева. Содержит: лого, навигацию, настройки аккаунта.
- Никаких новых API-эндпоинтов или таблиц — чисто фронтенд-изменение.

## Edge Cases

- Если у пользователя нет заведений — ссылка на расписание не показывается (нет slug).
- Telegram уже привязан — в бургере показывается статус «Telegram привязан», кнопка привязки всё равно доступна (можно перепривязать).
- На десктопе и мобильном бургер-меню работает одинаково (единообразный UX).
