# Фича: Яндекс Метрика

**Ветка:** `22-yandex-metrika`
**Создана:** 2026-03-17
**Статус:** реализовано

## Контекст

**Зачем:** Сейчас нет никакой аналитики по посещениям. Непонятно, сколько людей заходят на лендинг, сколько владельцев пользуются админкой. Яндекс Метрика — бесплатный инструмент с вебвизором, картой кликов и воронками.

**Что есть сейчас:** Две веб-поверхности без аналитики:
- **Лендинг** (`slotik.tech`) — статический HTML, точка привлечения новых пользователей
- **Админ-панель** (`admin.slotik.tech`) — React SPA (Vite), управление заведениями

**Решение:** Один счётчик Яндекс Метрики на обе поверхности. Метрика различает поддомены автоматически. Counter ID передаётся через переменные окружения, чтобы не хардкодить. Вебвизор включается для лендинга.

## User Stories

### US-1 — Аналитика лендинга (Приоритет: P1)

Как владелец продукта, я хочу видеть в Яндекс Метрике статистику посещений лендинга (визиты, источники трафика, вебвизор, карта кликов), чтобы оценивать эффективность лендинга и оптимизировать конверсию.

**Почему этот приоритет:** Лендинг — главная точка входа новых пользователей, его конверсия критична.

**Как проверить независимо:** Открыть `slotik.tech`, в DevTools Network видны запросы к `mc.yandex.ru`, визит появляется в Метрике.

**Сценарии приёмки:**

1. **Дано** пользователь открывает `slotik.tech`, **Когда** страница загружена, **Тогда** счётчик Яндекс Метрики инициализируется и отправляет данные о визите.
2. **Дано** вебвизор включен в настройках счётчика, **Когда** пользователь скроллит и кликает по лендингу, **Тогда** запись его сессии доступна в разделе «Вебвизор» в Метрике.

---

### US-2 — Аналитика админ-панели (Приоритет: P1)

Как владелец продукта, я хочу видеть статистику использования админ-панели, чтобы понимать активность владельцев заведений.

**Почему этот приоритет:** Нужно понимать, пользуются ли владельцы панелью после регистрации.

**Как проверить независимо:** Открыть `admin.slotik.tech`, в DevTools Network видны запросы к `mc.yandex.ru`, визит появляется в Метрике.

**Сценарии приёмки:**

1. **Дано** владелец заходит в админ-панель, **Когда** страница загружена, **Тогда** счётчик Яндекс Метрики фиксирует визит.
2. **Дано** админ-панель — React SPA, **Когда** владелец переключается между страницами (Чат / Календарь), **Тогда** Метрика корректно отслеживает переходы.

---

## Ключевые сущности

- **Счётчик Яндекс Метрики**: JS-код, встраиваемый на страницу, отправляет данные в Яндекс. Идентифицируется числовым counter ID.
- **Counter ID**: числовой идентификатор, создаётся в https://metrika.yandex.ru. Один счётчик на обе поверхности (лендинг + админка). Метрика различает домены автоматически.

## Edge Cases

- Если `METRIKA_COUNTER_ID` не задан — счётчик не встраивается, страницы работают как раньше.
- Dev-режим: никакой специальной логики — если переменная не задана (а в dev она не задана), Метрика просто не подключается.

## Критерии приёмки

- [x] На `slotik.tech` загружается счётчик Яндекс Метрики (запросы к `mc.yandex.ru` в Network)
- [x] На `admin.slotik.tech` загружается тот же счётчик
- [x] Если `METRIKA_COUNTER_ID` не задан — ни на лендинге, ни в админке код счётчика не вставляется
- [ ] В Метрике видны визиты с обоих доменов (проверить после деплоя)
- [x] Вебвизор включен на лендинге (`webvisor: true` в коде счётчика)
