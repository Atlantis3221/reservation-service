# Фича: Мультитенант на фронтенде

**Ветка:** `03-frontend-multitenant`
**Статус:** не начат

## Контекст

**Зачем:** клиент должен заходить на `domain.ru/slug` и видеть расписание конкретной бани с возможностью связаться с владельцем через Telegram.

**Что есть сейчас:** фронтенд — один SPA без роутинга, привязан к единственной бане. Есть мёртвый код: `ReservationForm`, `ReservationList`, CRUD-роуты `/api/reservations`.

## User Stories

### US-1 — Клиент видит расписание бани по slug (Приоритет: P1)

Клиент переходит по ссылке `domain.ru/banya-u-petra`, видит название бани, доступные даты и слоты. Может выбрать слот и написать владельцу в Telegram.

**Почему этот приоритет:** это основная ценность для конечного клиента — возможность увидеть свободные окна и записаться.

**Как проверить независимо:** открыть `domain.ru/slug` в браузере, увидеть название бани, выбрать дату, увидеть слоты, нажать кнопку «Написать в Telegram».

**Сценарии приёмки:**

1. **Дано** баня с slug `test-banya` зарегистрирована, **Когда** клиент открывает `/test-banya`, **Тогда** отображается название бани и календарь
2. **Дано** есть свободные слоты на завтра, **Когда** клиент выбирает завтрашнюю дату, **Тогда** отображаются свободные часы
3. **Дано** клиент выбрал слот, **Когда** он нажимает кнопку связи, **Тогда** открывается Telegram диалог с владельцем

---

### US-2 — Корневая страница-заглушка (Приоритет: P2)

На корневой странице `/` — информация о сервисе и инструкция как подключить свою баню.

**Сценарии приёмки:**

1. **Дано** пользователь открывает `/`, **Тогда** видит текст «Это сервис бронирования» и ссылку на бота

---

### US-3 — Несуществующий slug (Приоритет: P2)

Клиент переходит по несуществующему slug — видит понятное сообщение об ошибке.

**Сценарии приёмки:**

1. **Дано** slug `nonexistent` не зарегистрирован, **Когда** клиент открывает `/nonexistent`, **Тогда** видит «Баня не найдена»

## Ключевые сущности

- **API эндпоинты:**
  - `GET /api/business/:slug` → { name, telegram_username }
  - `GET /api/business/:slug/available-dates` → string[]
  - `GET /api/business/:slug/day-slots?date=YYYY-MM-DD` → SlotStatus[]

## Edge Cases

- Что если баня зарегистрирована, но у неё нет ни одного слота? — показать «Расписание пока не настроено»
- Что если у владельца нет Telegram username? — скрыть кнопку «Написать в Telegram», показать «Свяжитесь с владельцем»
