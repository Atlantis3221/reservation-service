# Фича: Демо-баня с ежедневными записями

**Ветка:** `26-demo-business`
**Создана:** 2026-03-18
**Статус:** реализовано

## Контекст

**Зачем:** На продакшене нужна тестовая баня с записями на текущий день, чтобы можно было демонстрировать работу сервиса — показывать календарь с реальными данными потенциальным клиентам и тестировать фичи на живом окружении.

**Что есть сейчас:** Демо-бизнеса нет. Все бизнесы создаются вручную через Telegram-бот или админ-панель. Нет механизма автоматического создания записей.

## User Stories

### US-1 — Автоматическое создание демо-бизнеса (Приоритет: P1)

При старте сервера система проверяет, существует ли бизнес со slug `demo-banya`. Если нет — создаёт его с именем «Демо Баня» и контактными ссылками.

**Почему этот приоритет:** Без бизнеса нечего показывать, это базовое условие.

**Как проверить независимо:** Запустить сервер на чистой БД → убедиться что бизнес `demo-banya` создан с контактами VK и Telegram.

**Сценарии приёмки:**

1. **Дано** чистая БД, **Когда** сервер стартует, **Тогда** создаётся бизнес slug=`demo-banya`, name=«Демо Баня»
2. **Дано** бизнес `demo-banya` уже существует, **Когда** сервер стартует, **Тогда** повторного создания не происходит
3. **Дано** бизнес создан, **Когда** запрашиваем контактные ссылки, **Тогда** возвращаются VK (`https://vk.com/ndrwbv`) и Telegram (`https://t.me/ndrwbv`)

---

### US-2 — Ежедневные записи на текущий день (Приоритет: P1)

Каждый день у демо-бани автоматически появляются 5 записей (booked) на текущий день. Записи создаются при старте сервера (если на сегодня их ещё нет) и по крону в полночь по МСК.

**Почему этот приоритет:** Главная цель фичи — всегда иметь актуальные записи для демонстрации.

**Как проверить независимо:** Запустить сервер → открыть календарь `demo-banya` → увидеть 5 записей на сегодня.

**Сценарии приёмки:**

1. **Дано** сервер запущен, на сегодня нет записей у демо-бани, **Когда** проходит инициализация, **Тогда** создаются 5 бронирований: 10:00–12:00, 13:00–15:00, 16:00–18:00, 19:00–21:00, 22:00–00:00
2. **Дано** на сегодня уже есть записи у демо-бани, **Когда** сервер перезапускается, **Тогда** повторного создания не происходит
3. **Дано** наступает новый день (00:05 MSK), **Когда** срабатывает крон, **Тогда** создаются записи на новый день
4. **Дано** записи на вчера и ранее, **Когда** срабатывает крон, **Тогда** старые записи демо-бани удаляются

## Ключевые сущности

- **Демо-бизнес**: строка в `businesses` со slug `demo-banya`, `owner_chat_id = 'demo'`. Не привязан к реальному Telegram-аккаунту.
- **Демо-записи**: строки в `slots` со `status = 'booked'`, привязаны к демо-бизнесу. Содержат фейковые имена клиентов и заметки для реалистичности.

## Edge Cases

- Если кто-то вручную удалит записи демо-бани на сегодня — они восстановятся только при следующем крон-запуске (полночь) или рестарте сервера
- Если кто-то отредактирует записи демо-бани — изменения сохранятся до следующего дня
- Демо-бизнес нельзя удалить через обычный UI (нет owner_chat_id привязанного к реальному пользователю)

## Критерии приёмки

- [ ] При старте сервера создаётся бизнес `demo-banya` если его нет
- [ ] У бизнеса есть контактные ссылки: VK `https://vk.com/ndrwbv`, Telegram `https://t.me/ndrwbv`
- [ ] При старте сервера создаются 5 записей на сегодня если их нет
- [ ] Записи: 10:00–12:00, 13:00–15:00, 16:00–18:00, 19:00–21:00, 22:00–00:00
- [ ] Записи имеют реалистичные имена клиентов и заметки
- [ ] Ежедневно в 00:05 MSK крон создаёт записи на новый день
- [ ] Крон удаляет записи демо-бани старше текущего дня
- [ ] Повторный запуск не создаёт дубликатов
