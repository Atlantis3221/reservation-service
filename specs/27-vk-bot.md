# Фича: Интеграция с ботом ВКонтакте

**Ветка:** `27-vk-bot`
**Создана:** 2026-03-18
**Статус:** уточнено

## Контекст

**Зачем:** Расширить каналы доступа к сервису — не все пользователи пользуются Telegram, VK по-прежнему популярен в России, особенно среди малого бизнеса. Бот ВК должен предоставлять полный функционал управления расписанием и бронированием, аналогичный Telegram-боту.

**Что есть сейчас:**
- Telegram-бот (Telegraf) с полным функционалом: регистрация заведения, расписание, бронирование, отмена, настройки, мультизаведения, пользовательское соглашение, ссылки для связи.
- Платформо-независимый слой `services/command.ts` — уже абстрагирует бизнес-логику от UI (используется админ-панелью).
- Парсеры (`bot/parsers.ts`) и форматтеры (`bot/formatters.ts`) не привязаны к Telegram API.

## User Stories

### US-1 — Регистрация заведения через VK-бота (Приоритет: P1)

Владелец заведения пишет VK-боту, принимает пользовательское соглашение, и регистрирует заведение — аналогично Telegram-боту.

**Почему этот приоритет:** Без регистрации невозможен весь остальной функционал.

**Как проверить независимо:** Написать боту ВК, пройти флоу соглашения и создания заведения, убедиться что заведение появилось в БД.

**Сценарии приёмки:**

1. **Дано** пользователь впервые пишет VK-боту, **Когда** отправляет любое сообщение, **Тогда** бот показывает пользовательское соглашение с кнопками «Принимаю» / «Отклоняю».
2. **Дано** пользователь принял соглашение, **Когда** бот спрашивает название, **Тогда** пользователь вводит название и подтверждает slug — заведение создаётся.
3. **Дано** пользователь отклонил соглашение, **Когда** он снова пишет боту, **Тогда** бот повторно предлагает соглашение.

---

### US-2 — Управление расписанием через VK-бота (Приоритет: P1)

Владелец задаёт время работы на неделю, просматривает расписание — теми же текстовыми командами, что и в Telegram.

**Почему этот приоритет:** Ключевой функционал сервиса.

**Сценарии приёмки:**

1. **Дано** заведение зарегистрировано, **Когда** владелец отправляет «на этой неделе пн-пт с 10 до 23», **Тогда** слоты создаются и бот отвечает сводкой.
2. **Дано** слоты заданы, **Когда** владелец отправляет «расписание на сегодня», **Тогда** бот показывает расписание дня.
3. **Дано** слоты заданы, **Когда** владелец отправляет «покажи расписание», **Тогда** бот показывает статистику на неделю.

---

### US-3 — Бронирование и отмена через VK-бота (Приоритет: P1)

Владелец создаёт и отменяет брони текстовыми командами.

**Сценарии приёмки:**

1. **Дано** расписание задано, **Когда** владелец отправляет «сегодня бронь с 14 до 18 Иванов», **Тогда** бот подтверждает бронирование с кнопкой отмены.
2. **Дано** есть пересечение с другой бронью, **Когда** владелец бронирует, **Тогда** бот спрашивает подтверждение.
3. **Дано** бронь существует, **Когда** владелец отправляет «отмени бронь на сегодня 14», **Тогда** бронь отменяется.

---

### US-4 — Управление заведениями через VK-бота (Приоритет: P2)

Мультизаведения: выбор, добавление, удаление, настройки.

**Сценарии приёмки:**

1. **Дано** у владельца несколько заведений, **Когда** он отправляет команду, **Тогда** бот предлагает выбрать заведение кнопками.
2. **Дано** заведение выбрано, **Когда** владелец отправляет «настройки», **Тогда** бот показывает настройки с кнопками изменения.

---

### US-5 — Привязка веб-панели и ссылки для связи (Приоритет: P2)

Команда `/link` генерирует код для привязки админ-панели. Управление контактными ссылками (Telegram, VK, MAX).

**Сценарии приёмки:**

1. **Дано** заведение зарегистрировано, **Когда** владелец отправляет «/link», **Тогда** бот возвращает 6-значный код привязки.
2. **Дано** заведение зарегистрировано, **Когда** владелец заходит в настройки → ссылки для связи, **Тогда** может добавить/удалить ссылки.

---

### US-6 — Кросс-платформенная привязка через админ-панель (Приоритет: P2)

Платформы (Telegram и VK) работают независимо — каждая создаёт свои заведения со своим `ownerChatId`. Связка между платформами происходит только через веб-панель: оба бота генерируют код `/link`, который привязывается к одному аккаунту в админке.

**Сценарии приёмки:**

1. **Дано** заведение создано через VK, **Когда** владелец отправляет `/link`, **Тогда** получает код для привязки к админ-панели.
2. **Дано** один аккаунт в админке привязан через Telegram, **Когда** владелец привязывает VK-бота через код, **Тогда** в админке видны заведения обеих платформ.

---

## Ключевые сущности

- **VK Bot**: Бот-сообщество ВКонтакте, получает сообщения через VK Bot Long Poll API.
- **ownerChatId**: Хранит ID владельца с платформенным префиксом: `tg:123456` для Telegram, `vk:789012` для VK. Обратная совместимость: существующие ID без префикса считаются Telegram.
- **Keyboard**: VK использует собственный формат клавиатур (inline / reply), отличный от Telegram Markup.

## Edge Cases

- **Телефон:** VK не поддерживает `request_contact` — для VK-пользователей шаг запроса телефона пропускается. VK-профиль сам по себе является каналом связи.
- **Форматирование:** VK не поддерживает HTML-разметку в сообщениях — форматтеры отдают plain-text (без `<b>`, `<code>` тегов).
- **Кнопки:** VK имеет лимиты на клавиатуру (максимум 10 кнопок, 5 строк) — адаптировать раскладку.
- **Long Poll:** Используем VK Bot Long Poll API (аналогично Telegram-боту). Не требует публичного URL.
- **Мониторинг:** VK-сообщения трекаются в `services/monitor.ts` наравне с Telegram.

## Критерии приёмки

- [ ] VK-бот запускается при наличии `VK_BOT_TOKEN` в .env, без токена — пропускается
- [ ] Полный флоу регистрации заведения работает через VK (соглашение → название → slug), без шага запроса телефона
- [ ] Все текстовые команды (бронирование, расписание, отмена, время работы) работают идентично Telegram
- [ ] Inline-кнопки (подтверждение, выбор заведения, настройки) работают через VK keyboard
- [ ] Мультизаведения с выбором через кнопки работают
- [ ] Команда /link генерирует код привязки к админ-панели
- [ ] Управление ссылками для связи работает
- [ ] Тексты адаптированы под VK (plain-text, без HTML-разметки)
- [ ] `ownerChatId` использует префиксы `vk:` / `tg:` для различения платформ
- [ ] Мониторинг-бот трекает сообщения VK-бота
