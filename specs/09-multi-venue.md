# Фича: Несколько бань у одного владельца

**Ветка:** `09-multi-venue`
**Статус:** черновик

## Контекст

**Зачем:** владелец может управлять несколькими банями. Бот должен корректно работать с выбором бани.

**Что есть сейчас:** таблица `businesses` уже поддерживает несколько записей, но `getBusinessByOwner(chatId)` возвращает только одну. Бот не умеет спрашивать «для какой бани?». Связь 1:1 — один `owner_chat_id` → один `business`.

## User Stories

### US-1 — Добавление бани (Приоритет: P1)

Владелец добавляет новую баню через бота (`/add_banya Баня на Озере banya-na-ozere`). Лимита на количество бань нет.

**Сценарии приёмки:**

1. **Дано** у владельца одна баня, **Когда** он отправляет `/add_banya Баня на Озере banya-na-ozere`, **Тогда** создаётся вторая баня с тем же `owner_chat_id`
2. **Дано** у владельца уже есть баня с таким slug, **Когда** он пытается добавить ещё одну с таким же slug, **Тогда** бот отвечает «Баня с таким идентификатором уже существует»

---

### US-2 — Автовыбор бани если она одна (Приоритет: P1)

Если у владельца одна баня — все команды (бронирование, расписание, отмена) работают без уточнения, как сейчас.

**Сценарии приёмки:**

1. **Дано** у владельца одна баня, **Когда** он отправляет команду бронирования, **Тогда** бот не спрашивает «для какой бани?» и работает как сейчас

---

### US-3 — Выбор бани через inline-кнопки (Приоритет: P1)

Если у владельца несколько бань и он отправляет команду — бот спрашивает «Для какой бани?» с inline-кнопками (по одной на каждую баню). Выбор только через кнопки, без указания в тексте.

**Сценарии приёмки:**

1. **Дано** у владельца 2 бани («Баня у Петра», «Баня на Озере»), **Когда** он пишет `сегодня бронь с 14 до 18 Олег`, **Тогда** бот спрашивает «Для какой бани?» с inline-кнопками
2. **Дано** владелец выбрал баню кнопкой, **Тогда** команда выполняется для выбранной бани

---

### US-4 — Удаление бани (Приоритет: P1)

Владелец удаляет баню через бота (`/del_banya banya-na-ozere`). Удаляются также все слоты этой бани.

**Сценарии приёмки:**

1. **Дано** у владельца 2 бани, **Когда** он пишет `/del_banya banya-na-ozere`, **Тогда** баня и все её слоты удаляются, бот подтверждает
2. **Дано** у владельца одна баня, **Когда** он пытается её удалить, **Тогда** бот отвечает «Нельзя удалить единственную баню»
3. **Дано** slug не принадлежит владельцу, **Тогда** бот отвечает «Баня не найдена»

---

### US-5 — Список бань владельца (Приоритет: P2)

Владелец пишет `/mybanya` — бот показывает список его бань.

**Сценарии приёмки:**

1. **Дано** у владельца 2 бани, **Когда** он пишет `/mybanya`, **Тогда** бот выводит список с названиями и slug

## Edge Cases

- Удаление бани с существующими будущими бронями — удалять вместе со слотами, без дополнительного предупреждения
- Фронтенд не меняется — каждая баня уже имеет свой slug и отдельный URL

## Критерии приёмки

- [ ] `getBusinessByOwner` → `getBusinessesByOwner` возвращает массив
- [ ] Если баня одна — поведение бота не меняется
- [ ] Если бань несколько — бот спрашивает «для какой?» через inline-кнопки
- [ ] Добавление новой бани через `/add_banya`
- [ ] Удаление бани через `/del_banya` (нельзя удалить единственную)
- [ ] Список бань владельца через `/mybanya`
- [ ] Фронтенд не затрагивается
