# Фича: Несколько заведений у одного владельца

**Ветка:** `09-multi-venue`
**Статус:** реализовано

## Контекст

**Зачем:** владелец может управлять несколькими заведениями. Бот должен корректно работать с выбором заведения.

**Что есть сейчас:** таблица `businesses` уже поддерживает несколько записей, но `getBusinessByOwner(chatId)` возвращает только одну. Бот не умеет спрашивать «для какого заведения?». Связь 1:1 — один `owner_chat_id` → один `business`.

## User Stories

### US-1 — Добавление заведения (Приоритет: P1)

Владелец добавляет новое заведение через бота (`/add`). Лимита на количество нет.

**Сценарии приёмки:**

1. **Дано** у владельца одно заведение, **Когда** он отправляет `/add`, **Тогда** бот запрашивает название и создаёт второе заведение с тем же `owner_chat_id`
2. **Дано** у владельца уже есть заведение с таким slug, **Когда** он пытается добавить ещё одно с таким же slug, **Тогда** бот отвечает «Slug уже занят»

---

### US-2 — Автовыбор если заведение одно (Приоритет: P1)

Если у владельца одно заведение — все команды работают без уточнения, как сейчас.

**Сценарии приёмки:**

1. **Дано** у владельца одно заведение, **Когда** он отправляет команду, **Тогда** бот не спрашивает «для какого?» и работает как сейчас

---

### US-3 — Выбор заведения через inline-кнопки (Приоритет: P1)

Если у владельца несколько заведений и он отправляет команду — бот спрашивает «Для какого заведения?» с inline-кнопками. Выбор только через кнопки.

**Сценарии приёмки:**

1. **Дано** у владельца 2 заведения, **Когда** он пишет `сегодня бронь с 14 до 18 Олег`, **Тогда** бот спрашивает «Для какого заведения?» с inline-кнопками
2. **Дано** владелец выбрал заведение кнопкой, **Тогда** команда выполняется для выбранного заведения

---

### US-4 — Удаление заведения (Приоритет: P1)

Владелец удаляет заведение через бота (`/del slug`). Удаляются также все слоты.

**Сценарии приёмки:**

1. **Дано** у владельца 2 заведения, **Когда** он пишет `/del slug`, **Тогда** заведение и все его слоты удаляются
2. **Дано** у владельца одно заведение, **Когда** он пытается удалить, **Тогда** бот отвечает «Нельзя удалить единственное заведение»
3. **Дано** slug не принадлежит владельцу, **Тогда** бот отвечает «Заведение не найдено»

---

### US-5 — Список заведений (Приоритет: P2)

Владелец пишет `/list` — бот показывает список его заведений.

**Сценарии приёмки:**

1. **Дано** у владельца 2 заведения, **Когда** он пишет `/list`, **Тогда** бот выводит список с названиями и slug

## Edge Cases

- Удаление заведения с существующими будущими бронями — удалять вместе со слотами, без дополнительного предупреждения
- Фронтенд не меняется — каждое заведение уже имеет свой slug и отдельный URL

## Критерии приёмки

- [x] `getBusinessByOwner` → `getBusinessesByOwner` возвращает массив
- [x] Если заведение одно — поведение бота не меняется
- [x] Если заведений несколько — бот спрашивает «для какого?» через inline-кнопки
- [x] Добавление нового заведения через `/add`
- [x] Удаление заведения через `/del` (нельзя удалить единственное)
- [x] Список заведений через `/list`
- [x] Фронтенд не затрагивается
